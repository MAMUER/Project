# ТЕХНИЧЕСКОЕ ЗАДАНИЕ  
на разработку программы «Веб-приложение для автоматического создания и управления программами индивидуальных тренировок на базе спортивного центра»

## 1. ВВЕДЕНИЕ

Наименование программы: **Веб-приложение для автоматического создания и управления программами индивидуальных тренировок на базе спортивного центра**.  
Программа предназначена для автоматизации процессов фитнес-центра, управления клиентской базой, расписанием занятий, оборудованием и персоналом.

Тема разработки: **fitness-club-management-system v1.0**.

## 3. НАЗНАЧЕНИЕ РАЗРАБОТКИ

**Функциональное назначение:**  
Автоматизация работы фитнес-центра, включая создание индивидуальных тренировочных программ, управление расписанием, оборудованием, клиентами и персоналом.

**Эксплуатационное назначение:**  
Использование в спортивных центрах для повышения эффективности управления услугами и улучшения клиентского опыта.

**Обозначение программы:** fitness-club-management-system v1.0

## 2. Функциональное назначение

**Класс решаемых задач:** Автоматизация процессов фитнес-центра и управление программами тренировок

**Назначение программы:**
- Создание индивидуальных программ тренировок на основе оборудования спортивного центра
- Управление клиентской базой и расписанием занятий
- Администрирование оборудования и возможностей фитнес-клубов
- Организация записи на тренировки и персональные занятия
- Управление персоналом (тренеры, административный staff)

**Функциональные ограничения на применение:**
- Регистрация доступна только пользователям старше 18 лет
- Доступ к функционалу ограничен в соответствии с ролью пользователя
- Для работы требуется база данных PostgreSQL
- Поддерживается одновременная работа до 100 пользователей

## 3. Описание логической структуры

**Алгоритм программы:**
Пользователь входит в систему → система проверяет роль → предоставление доступа к модулям:
- MEMBER: запись на тренировки, просмотр программ, личный кабинет
- TRAINER: управление расписанием, проведение тренировок
- STAFF: администрирование клубов, управление оборудованием

**Используемые методы:**
- Spring Security для аутентификации и авторизации
- Spring Data JPA для работы с базой данных
- MVC архитектура для веб-интерфейса
- BCrypt для хеширования паролей
- Adaptive programming для генерации тренировочных программ

**Структура программы:**
```
com.example.project/
├── config/ - Конфигурационные классы
├── controller/ - Веб-контроллеры
├── model/ - Сущности базы данных
├── repository/ - Репозитории для работы с БД
├── service/ - Бизнес-логика
├── dto/ - Data Transfer Objects
└── resources/
    ├── templates/ - HTML шаблоны
    ├── static/ - CSS, JS, изображения
    └── database/ - SQL скрипты
```

**Основные модули:**
- `MainController` - основной контроллер приложения
- `ClubAdminController` - управление клубами
- `EquipmentAdminController` - управление оборудованием
- `CustomUserDetailsService` - управление пользователями
- `AdaptiveProgramGenerator` - генератор программ тренировок

**Связи программы с другими программами:**
- Интеграция с базой данных PostgreSQL
- Веб-сервер Spring Boot Embedded Tomcat

### Примеры использования
Запуск в среде разработки:

Установите PostgreSQL и создайте базу данных 'fitness_club_db'

Настройте application.properties:
- spring.datasource.url=jdbc:postgresql://postgres:5432/fitness_club_db
- spring.datasource.username=your_username
- spring.datasource.password=

Запустите приложение:
mvn spring-boot:run


Доступ к приложению:

* Откройте браузер и перейдите по адресу http://localhost:8080
* Войдите используя тестовые учетные записи
* В зависимости от роли получите доступ к соответствующему функционалу
