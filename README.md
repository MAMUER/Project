# FitPulse — Интеллектуальная платформа персонализированных тренировок

---

## Технические сведения о программе (по ГОСТ 19.102-77)

### 1. Общие сведения

**Полное наименование:** FitPulse — интеллектуальная веб-платформа для персонализированного управления физической активностью, прогнозирования рисков и консультаций с врачом на основе анализа биометрических данных.

**Краткое наименование:** FitPulse.

**Версия программы:** 2.0.0.

**Платформа:**
- Серверная часть: Linux (Docker, Kubernetes), Windows (разработка)
- Клиентская часть: мобильный веб-браузер (Chrome, Firefox, Edge, Safari)

**Языки разработки:** Go 1.25, Python 3.11, JavaScript (ES2022), SQL.

### 2. Функциональное назначение

Программа предназначена для:
- регистрации и авторизации пользователей с подтверждением email (SMTP);
- отдельной регистрации врачей и администраторов через **invite-коды**;
- сбора и хранения биометрических данных с носимых устройств (Apple Watch, Samsung Galaxy Watch, Huawei Watch D2, Amazfit T-Rex 3);
- эмуляции носимых устройств при отсутствии реальных (реалистичная генерация пульса, ЭКГ, давления, SpO₂, температуры, сна, шагов, HRV);
- классификации текущего состояния пользователя с помощью ML-модели (6 классов: восстановление, выносливость E1-E2, пороговая E3, силовая/HIIT, перетренированность, заболевание);
- генерации индивидуальных тренировочных планов с использованием GAN;
- генерации рекомендованной диеты с расчётом макронутриентов;
- ежедневной адаптивной модификации плана тренировок при отклонении биометрических показателей;
- консультаций с персональным врачом (чат, назначения, редактирование тренировок);
- управления профилем пользователя (возраст, вес, рост, цели, противопоказания, питание, сон);
- мониторинга состояния здоровья в реальном времени;
- отслеживания достижений и прогресса тренировок.

### 3. Описание логической структуры

Система построена по микросервисной архитектуре. Взаимодействие компонентов:

```
┌──────────────┐
│   Browser    │  HTTPS (8443) — мобильное веб-приложение (6 вкладок)
└──────┬───────┘
       │
┌──────▼───────┐
│  NGINX (LB)  │  SSL termination, CSP, rate limiting, server_tokens off
└──────┬───────┘
       │
┌──────▼───────┐
│   Gateway    │  HTTP REST → gRPC proxy (8080) — единая точка входа
└──┬───┬───┬───┘
   │   │   │
   ▼   ▼   ▼
┌─────┐ ┌──────┐ ┌────────┐ ┌────────┐
│User │ │Bio   │ │Training│ │Doctor  │  gRPC services
│Svc  │ │Svc   │ │Svc     │ │Svc     │
└──┬──┘ └──┬───┘ └───┬────┘ └───┬────┘
   │       │         │          │
   ▼       ▼         ▼          ▼
┌──────────────────────────────────────┐
│           PostgreSQL 15              │
└──────────────────────────────────────┘
       │              │              │
   ┌───▼───┐    ┌────▼────┐   ┌────▼─────┐
   │ Redis │    │RabbitMQ │   │ML Engines│
   │   7   │    │  3.12   │   │Classifier│
   └───────┘    └────┬────┘   │+ GAN     │
                     │        └──────────┘
              ┌──────▼──────┐
              │  Device     │
              │  Connector  │  ← Apple Watch, Samsung, Huawei, Amazfit
              │  + Emulator │  (эмуляция реальных протоколов)
              └─────────────┘
```

**Компоненты системы:**

| Компонент | Технология | Порт | Назначение |
|-----------|-----------|------|------------|
| Gateway | Go, gorilla/mux | 8080 | HTTP REST API, проксирование к gRPC |
| User Service | Go, gRPC | 50051 | Регистрация, авторизация, профили, email-верификация, invite-коды |
| Biometric Service | Go, gRPC | 50052 | Сбор и хранение биометрических данных |
| Training Service | Go, gRPC | 50053 | Управление тренировочными планами |
| Doctor Service | Go, gRPC | 50054 | Чат, назначения, модификация тренировок, подписки |
| Device Connector | Go, HTTP | 8082 | Подключение внешних устройств |
| Device Emulator | Go | — | Эмуляция носимых устройств |
| ML Classifier | Python, FastAPI | 8001 | Классификация состояния (6 классов) |
| ML Generator | Python, FastAPI | 8002 | Генерация планов (GAN) |
| NGINX LB | Nginx, Alpine | 8443 | SSL termination, балансировка, CSP |
| PostgreSQL | 15 | 5432 | Основная база данных |
| Redis | 7 | 6379 | Кэширование, сессии, auth-коды |
| RabbitMQ | 3.12 | 5672 | Асинхронный обмен (ML jobs, биометрия) |

### 4. Описание взаимодействия с другими компонентами

**Внешние зависимости:**
- SMTP-сервер (Yandex Mail, Mail.ru, Gmail) — отправка писем верификации
- Веб-браузер клиента — отображение мобильного интерфейса
- Носимые устройства (Apple Watch, Samsung Galaxy Watch, Huawei Watch D2, Amazfit T-Rex 3) — биометрия

**Протоколы взаимодействия:**

- Микросервисы ↔ Redis: RESP

### 5. Настройка и установка

#### Требования
- Docker 24+ / Docker Compose v2
- Go 1.25+ (для локальной разработки)
- Python 3.11+ (для ML-сервисов)
- 4 ГБ ОЗУ минимум

#### Развёртывание (Docker Compose)

```bash
# 1. Скопировать и настроить .env
cp .env.example .env   # заполнить SMTP, JWT_SECRET, пароли БД

# 2. Запустить все сервисы
docker compose -f deployments/docker-compose.yml up -d

# 3. Проверить работоспособность
docker compose -f deployments/docker-compose.yml ps
curl -k https://localhost:8443/health
```

#### Развёртывание (Kubernetes)

```bash
minikube start
kubectl apply -f configs/k8s/namespace.yaml
kubectl apply -f configs/k8s/configmap.yaml
kubectl apply -f configs/k8s/deployments/
kubectl apply -f configs/k8s/services/
kubectl get pods -n fitness-platform -w
```

#### Настройка SMTP (Yandex)

1. Создать пароль приложения: https://passport.yandex.ru/profile → Безопасность → Пароли приложений
2. В `.env` указать:
   ```
   SMTP_HOST=smtp.yandex.ru
   SMTP_PORT=465
   SMTP_USER=ваш_email@yandex.ru
   SMTP_PASSWORD=пароль_приложения
   SMTP_FROM=ваш_email@yandex.ru
   SMTP_TLS=true
   ```

#### Регистрация первого администратора

```sql
-- Выполнить в БД:
SELECT create_invite_code('admin', NULL, 1, 365);
-- Вернёт код вида: ADMIN-2026-<hash>
```

Затем зарегистрироваться через `POST /api/v1/register/invite` с этим кодом.

#### Запуск эмулятора устройств

```bash
go run ./cmd/device-emulator \
  --user-id=<USER_ID> \
  --device-type=huawei_watch_d2 \
  --connector-url=http://localhost:8082 \
  --sync-interval=30s
```

### 6. Входные и выходные данные

**Входные данные:**
- Данные пользователя: email, пароль, ФИО, роль (client, doctor, admin)
- Invite-код для регистрации врача/админа
- Биометрические данные: тип метрики (heart_rate, spo2, temperature, ecg, blood_pressure, sleep, steps, hrv), значение, timestamp, устройство
- Параметры профиля: возраст, пол, рост, вес, уровень подготовки, цели, противопоказания, питание, сон
- Параметры тренировки: длительность (недели), доступные дни, время суток, место (дом/зал/бассейн/улица)
- Данные здоровья: заболевания, особенности тренировок
- Сообщения чата с врачом

**Выходные данные:**
- JWT access token (HS256, TTL 24 ч)
- Тренировочный план (JSON: упражнения, подходы, повторения, длительность, расписание по дням и времени суток)
- Рекомендованная диета (JSON: приёмы пищи, калории, макронутриенты)
- Классификация состояния (JSON: класс, уверенность, рекомендации)
- Результаты чата с врачом (сообщения, назначения)
- Письмо верификации email (HTML)
- HMAC-SHA256 подпись критических ответов

### 1. Наименование и область применения

**1.2.** Область применения: автоматизация планирования тренировок, мониторинг состояния здоровья, профилактика перетренированности и травм, генерация диеты, консультантская поддержка врача.

### 2. Основание для разработки

**2.2.** Цель разработки: создание масштабируемой платформы, обеспечивающей индивидуальный подбор тренировочных нагрузок и диеты на основе биометрического анализа с носимых устройств, а также возможность консультаций с врачом.

### 3. Назначение разработки

**3.1.** Программа предназначена для:

**Для пользователя:**
- регистрации и аутентификации с двухфакторной верификацией email;
- подключения носимых устройств (Apple Watch, Samsung Galaxy Watch, Huawei Watch D2, Amazfit T-Rex 3) или эмуляции при отсутствии реальных;
- непрерывного сбора биометрических показателей (ЧСС, ЭКГ, давление, сатурация, температура, сон, шаги, HRV);
- автоматической классификации текущего состояния пользователя (6 классов);
- генерации адаптивных тренировочных планов с применением GAN (расписание по дням, упражнениям, времени суток);
- генерации рекомендованной диеты с расчётом макронутриентов;
- ежедневной адаптивной модификации плана при отклонении показателей;
- ведения истории тренировок и отслеживания прогресса;
- консультаций с персональным врачом (при наличии подписки);

**Для врача:**
- регистрации через invite-код (номер лицензии, специальность);

- создания назначений (рекомендации, изменение диеты, изменение тренировок, медикаменты);
- редактирования тренировочного плана пациента;
- планирования консультаций;
- просмотра истории модификаций тренировок.

**Для администратора:**
- регистрации через invite-код;
- управления пользователями;
- создания invite-кодов для новых врачей и администраторов.

**3.2.** Эксплуатация: мобильное веб-приложение, доступное через браузер (6 вкладок: Обзор, Профиль, Тренировки, Устройства, Врач, Диета).

#### 4.1. Требования к функциональным характеристикам

- регистрация → верификация email → логин → получение профиля работают последовательно;
- регистрация врача через invite-код → чат → назначения работают;
- эмулятор устройств подключается и отправляет биометрию.

## Система invite-кодов

Вместо открытой регистрации врачи и администраторы регистрируются через **уникальные коды приглашения**:

| Код | Роль | Специальность | Лимит |
|-----|------|--------------|-------|
| `DOCTOR-2026-SPORT-MED-001` | doctor | sports_medicine | 1 |
| `DOCTOR-2026-REHAB-002` | doctor | rehabilitation | 1 |
| `DOCTOR-2026-GENERAL-003` | doctor | general_practice | 5 |
| `ADMIN-2026-SETUP-ROOT-001` | admin | — | 1 |

API для валидации: `POST /api/v1/invite/validate`
Регистрация: `POST /api/v1/register/invite`

## ML/GAN комплексная логика

### Классификация состояния (TrainingStateClassifier)

| Состояние | Описание | Рекомендации |
|-----------|----------|--------------|
| **Восстановление** | Низкий пульс, высокий HRV | Лёгкая активность, растяжка, йога |
| **Базовая выносливость (E1-E2)** | Пульс 65-80% HRmax | Бег/велосипед 45-90 мин |
| **Пороговая выносливость (E3)** | Пульс 80-90% HRmax | Темповые интервалы 30-60 мин |
| **Силовая/HIIT** | Пульс 90-100% HRmax | HIIT, силовые, спринты |
| **Перетренированность** | Низкий HRV + высокий пульс | Отдых 1-3 дня |
| **Заболевание** | Температура > 37.5°C | Прекратить тренировки |

### Генерация тренировок и диеты

Адаптивный план на основе: состояния пользователя, цели, места (дом/зал/бассейн/улица), доступных дней и времени суток, подключённых устройств.

Диета: расчёт калорий по формуле Миффлина-Сан Жеора, типы диет (сбалансированная, высокобелковая, для похудения), макронутриенты, приёмы пищи по времени.

## Security

| # | Мера | Реализация |
|---|------|-----------|
| 1 | Инвалидация сессии | `LogoutHeaders()` очищает cookies, удаление сессии из Redis |
| 2 | Однократные auth-коды | OAuth2 коды, удаление после использования |
| 3 | Критические сессии | Отдельные Redis ключи, TTL 15 мин |
| 4 | 403 → 404 | Все forbidden ответы возвращают 404 |
| 5 | Маскировка версий | NGINX `server_tokens off`, middleware удаляет заголовки |
| 6 | Кастомные ошибки | HTML страницы без технической информации |
| 7 | Принудительное завершение | `return` после каждого `http.Error()` |
| 8 | Pre-handler middleware | Auth/privilege middleware до обработчика |
| 9 | Единая точка входа | Gateway для всех запросов |
| 10 | Серверная проверка ролей | Прямой запрос к БД при привилегированных операциях |
| 11 | Подпись ответов | HMAC-SHA256 для критических JSON |
| 12 | Content Security Policy | Строгая CSP с ограничением скриптов |
| 13 | bcrypt | Хеширование паролей (cost 10) |
| 14 | Rate limiting | NGINX + Go per-IP (10 r/s, burst 20) |

## CI/CD Pipeline

```
push/PR → test → security-scan → secrets-scan → build → docker → deploy → notify
```

- **test**: go vet, golangci-lint, go test
- **security-scan**: gosec (SAST), govulncheck, Trivy
- **secrets-scan**: TruffleHog, Gitleaks
- **build**: компиляция Go бинарников
- **docker**: сборка и push Docker образов
- **deploy**: VPS (Docker Compose) + Kubernetes
- **notify**: Telegram с полным отчётом
